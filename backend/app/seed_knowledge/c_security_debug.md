# C/C++ 网络设备代码 Debug 与安全规则

## 内存与资源
发现段错误、OOM 或进程异常退出时，优先获取 backtrace 和触发输入。检查空指针、数组边界、use-after-free、double free、整数溢出和错误路径资源释放。候选补丁必须经过重新编译、静态分析、单元测试和必要的 Sanitizer 验证。

## 字符串和配置输入
来自日志、Web、TR-069、OMCI、CLI 或配置文件的字符串均视为不可信输入。固定缓冲区写入应检查长度和截断，`snprintf` 也必须检查返回值。配置范围应依据设备频段、地区码、产品能力和协议约束，不能套用通用常量。

## 自动修复边界
系统生成的补丁只能作为候选 diff，默认不得覆盖原始代码。若缺少完整调用关系、宏定义、平台分支或业务规则，应标记 `NEED_HUMAN_REVIEW`。
