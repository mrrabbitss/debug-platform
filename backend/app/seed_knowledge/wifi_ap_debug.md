# AP 与 Wi-Fi 故障诊断规则

## hostapd 启动或重载失败
当日志出现 `Failed to set beacon parameters`、`Could not configure driver` 或 hostapd 启动失败时，应依次核对无线接口是否存在、国家码与信道是否合法、带宽和频段能力是否匹配、加密配置是否完整，以及驱动是否已经初始化。不能仅凭一条 hostapd 日志断言驱动故障，应结合内核和接口状态日志。

## 认证与四次握手失败
认证失败需要区分开放认证、WPA/WPA2/WPA3、802.1X/EAP 以及终端兼容问题。建议关联终端 MAC 脱敏标识、认证阶段、失败码、时间同步和密钥配置。若日志只出现客户端主动 deauth，不能直接判断 AP 侧异常。

## WLAN 状态变化
无线接口 down、hostapd 重启和客户端断连可能构成故障传播链，但时间相邻不等于因果关系。应确认配置下发、驱动告警或进程退出等前置事件。
